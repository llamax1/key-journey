Press A

|
  1. You pressed A
  2. A microchip in the keyboard detects which key has been pressed, and generates the appropriate ASCII code (01000001 for A)
  3. This is sent through the cable, into the USB (or other interface, such as PS/2 or perhaps SPI or I2C on an embedded system) port.
  4. It triggers an Interrupt Service routine, and is moved to the keyboard buffer in the RAM
  5. The CPU processes the input as per the programme's code, if there is anything that this input does
  6. The CPU always uses the FDE cycle
    1. The programme conter contents are copied to the Memory Address Register and the PC is set to point ot the next instruction.
    2. The instruction or data at the address in the MAR is copied to the MDR.
    3. The control unit decodes the instruction
    4. Calculations are performed by the Arithmetic and Logic Unit, with the results stored in the accumulator
